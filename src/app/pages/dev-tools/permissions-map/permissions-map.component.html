Critical: /admin Guard

The current adminGuard only checks if a user is authenticated. It does NOT enforce role checks. Any logged-in user (including CUSTOMERS) can hypothetically access the root /admin layout, though they might be blocked from inner pages or API calls.

Role Heirarchy

SUPER_ADMIN has universal access. MANAGER is optimized for Command Center. OPERATIONS is strictly limited to the Operations Portal.
